# 需求规格说明书

> 阶段 1（需求分析）产出。项目：博客系统（W 模型端到端调测）。

## 文档信息

- 项目名称：博客系统（blog-system-demo）
- 文档版本：v1.0
- 编制日期：2026-07-20
- 编制者：W-Model Agent（self-as-verifier 调测模式）

## 1. 项目概述

### 1.1 项目背景

需要从零交付一个博客系统后端 API，用于支撑前端博客站点的文章发布、公开浏览与互动。本次为 W 模型端到端调测，要求 8 阶段全流程闭环、产物可执行、质量门通过。

### 1.2 项目目标

- 交付可运行的博客系统后端（REST API）
- 验证 W 模型 8 阶段编排 + LLM-as-a-Verifier 阶段门 + 工件质量门端到端可用
- 单元测试覆盖率 ≥ 80%、RTM 需求覆盖率 100%、四级测试全部通过

### 1.3 范围

- 包含：用户认证（注册/登录/JWT）、文章 CRUD（作者隔离）、文章公开浏览、文章评论
- 不包含：前端 UI、富文本编辑器、文件上传、邮件通知、管理后台、软删除/版本管理

## 2. 需求清单

### 2.1 功能需求

| 需求 ID | 模块 | 需求描述 | 优先级 | 验收标准 |
|---|---|---|---|---|
| REQ-001 | 用户认证 | 用户可注册账号（用户名+密码），登录后获取 JWT；后续需认证接口通过 Bearer Token 鉴权 | 高 | 注册返回 201 + userId；重复用户名返回 409；登录成功返回 200 + JWT；错误密码返回 401；缺失/无效 token 返回 401 |
| REQ-002 | 文章管理 | 已认证用户可创建/编辑/删除自己的文章；非作者不可改他人文章 | 高 | 创建返回 201 + articleId；未认证返回 401；作者编辑返回 200；非作者编辑返回 403；删除返回 204 |
| REQ-003 | 公开浏览 | 任何人（无需认证）可浏览文章列表与文章详情 | 高 | 列表返回 200 + 数组；详情返回 200 + 文章内容；不存在的文章返回 404 |
| REQ-004 | 评论 | 已认证用户可对任意已存在文章发表评论；任何人可查看文章评论列表 | 中 | 已认证评论返回 201 + commentId；未认证评论返回 401；对不存在文章评论返回 404；查看评论列表返回 200 + 数组 |

### 2.2 非功能需求

| 需求 ID | 类别 | 描述 | 指标 |
|---|---|---|---|
| NFR-001 | 安全 | 密码必须 bcrypt 哈希后存储；JWT 过期时间 1 小时；明文密码不得出现在任何日志/响应 | bcrypt cost ≥ 10；JWT exp ≤ 3600s |
| NFR-002 | 性能 | 单接口 P95 响应时间 | ≤ 200ms（内存存储） |
| NFR-003 | 可维护性 | TypeScript 严格模式；所有公共 API 输入用 zod 校验 | tsc --strict 0 错误 |
| NFR-004 | 可测试性 | 单元测试覆盖率 | ≥ 80% |

### 2.3 约束需求

- 技术约束：Node.js ≥20 / Express 4 / TypeScript 5 / 内存存储（无外部 DB 依赖，便于端到端调测）
- 端口约束：默认监听 3000
- 数据约束：内存存储重启后数据丢失（已知限制，不视为缺陷）

## 3. 需求完整性检查

| 检查项 | 状态 | 说明 |
|---|---|---|
| 功能需求闭环 | ✅ | 认证 → 写文章 → 浏览 → 评论 形成闭环；每个 REQ 均有验收标准 |
| 非功能需求覆盖 | ✅ | 安全/性能/可维护性/可测试性 均有指标 |
| 冲突检测 | ✅ | 公开浏览（无需认证）与文章管理（需认证+作者隔离）无冲突：浏览对象限定为已发布文章的只读视图 |

## 4. 需求风险评估

| 风险 ID | 风险描述 | 等级 | 缓解措施 |
|---|---|---|---|
| RISK-001 | 内存存储重启数据丢失 | 中 | 文档显式标注为已知限制；后续可替换为 SQLite 不改 API |
| RISK-002 | JWT 密钥硬编码风险 | 中 | 通过环境变量 `JWT_SECRET` 注入；缺省时启动报错 |
| RISK-003 | 并发写入竞态（Map 非线程安全） | 低 | Node 单线程事件循环，业务量下不触发；文档标注 |
| RISK-004 | 作者隔离逻辑误判（token vs 文章作者） | 高 | 单元测试覆盖「非作者 403」场景；集成测试二次验证 |
