## 用例列表

### UAT-001

- 标题：用户注册成功（正常路径）
- 优先级：高
- 关联需求/设计：REQ-002
- 测试场景：用户使用合法用户名与密码注册，应成功创建账号并返回 201

**前置条件**

- 服务已启动（`npm run dev`）
- 用户 `alice` 不存在

**测试步骤**

| 步骤 | 操作 | 输入 | 预期输出 |
|---|---|---|---|
| 1 | POST /api/auth/register | `{"username":"alice","password":"Secret123"}` | HTTP 201，响应体含 `userId` 字段，不含明文密码 |
| 2 | 验证密码哈希存储 | 查询内存 Map 中 alice 记录 | `passwordHash` 字段以 `$2b$` 或 `$2a$` 开头（bcrypt 哈希），非明文 |

**预期结果**

注册返回 201，`userId` 为非空字符串；存储层中 `alice` 的密码以 bcrypt 哈希形式保存。

**执行状态**

- [ ] 待执行

### UAT-010

- 标题：普通用户列表查询不返回 rejected 文章（边界）
- 优先级：高
- 关联需求/设计：REQ-005
- 测试场景：普通用户查询文章列表，结果中不应包含被驳回的文章

**前置条件**

- 服务已启动
- 用户 `alice` 持有 JWT
- 文章 `art-001` status=approved，文章 `art-002` status=rejected（管理员已驳回）

**测试步骤**

| 步骤 | 操作 | 输入 | 预期输出 |
|---|---|---|---|
| 1 | GET /api/articles | Header: `Authorization: Bearer ${aliceToken}` | HTTP 200，响应体为数组，含 art-001 但 **不含** art-002 |
| 2 | GET /api/articles/art-002 | Header: `Authorization: Bearer ${aliceToken}` | HTTP 404，文章对普通用户不可见 |

**预期结果**

普通用户列表查询仅返回 approved 文章；rejected 文章详情查询返回 404。

**执行状态**

- [ ] 待执行
