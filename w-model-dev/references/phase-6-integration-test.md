# 阶段 6：集成测试（执行）

> W 模型右 V 测试执行阶段。设计来源：阶段 3（概要设计）产出的集成测试用例。
> 命令入口：`/wm test type=集成`

## 功能描述

验证模块间的交互正确性。执行阶段 3 设计的集成测试用例，确认接口契约、参数校验、跨模块数据传递符合设计。

## 输入

- 集成测试设计文档（阶段 3 产出）
- 已完成的模块代码（阶段 5 产出）

## 输出

- 集成测试执行结果（套用 [templates/test-report.md](../templates/test-report.md)，类型=集成测试）
- 接口兼容性报告

## AI 能力应用

- **集成测试用例执行**：按设计用例调用接口
- **接口调用验证**：合法 / 非法参数、状态码、返回结构
- **测试结果分析**：定位失败根因，关联到模块

## 测试用例设计（执行）

| 用例 ID | 测试场景 | 输入 | 预期输出 | 优先级 |
|---|---|---|---|---|
| IT-001 | 接口调用验证 | 合法请求参数 | 返回预期结果，状态码 200 | 高 |
| IT-002 | 接口参数校验 | 非法参数 | 返回错误信息，状态码 400 | 高 |
| IT-003 | 模块间数据传递 | 跨模块调用 | 数据正确传递和处理 | 高 |
| IT-004 | 接口性能测试 | 高并发请求 | 响应时间 < 500ms | 中 |
| IT-005 | 接口兼容性 | 不同版本接口 | 向后兼容或提示版本升级 | 中 |

## RTM 登记

在 [templates/rtm.md](../templates/rtm.md) 中更新：集成测试列的状态（通过 / 失败）。RTM 维护规则见 [rtm-guide.md](rtm-guide.md)。

## 验收标准

- [ ] 所有接口调用验证通过
- [ ] 参数校验逻辑正确
- [ ] 模块间数据传递无误
- [ ] 接口性能满足要求（响应时间 < 500ms）
- [ ] 失败用例已定位根因并回归

## 阶段门评审

集成测试全部通过 → 进入阶段 7（系统测试）。
不通过 → 回到编码实现返工，修复后重跑集成测试。

## 退出状态

项目 `status` 更新为 `系统测试`。
